# Contributing

Thank you for helping improve AI Team Room.

1. Open an issue before a large behavioral change.
2. Keep the server local-first and provider-neutral.
3. Do not add automatic permission escalation, hidden terminal injection, or transcript upload.
4. Add tests for protocol and security changes.
5. Run `python -m unittest discover -s tests -v` before submitting a pull request.
6. Keep commits focused and describe user-visible behavior and security implications.

Bug reports should include the OS, Python version, exact command, expected behavior, actual behavior, and a minimal reproduction with secrets removed.
